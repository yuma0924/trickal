Rank-lab プロジェクト統合仕様書

1. サイト基本定義
- 名称: Rank-lab（ランク・ラボ）※ドメイン・システム名称
- サイト表示名: みんなで決めるトリッカルランキング
- ドメイン: rank-lab.com
- 対象タイトル: 『トリッカル・もちもちほっぺ大作戦』
- コンセプト: トリッカルのキャラクター性能データと、投票・コメントによるランキング/掲示板を統合した非公式データベースサイト
- キャッチコピー: 投票やコメントでみんなの評価が見える
- 運用形態:
  - ログイン不要
  - 1人1票制（同一キャラへの投票コメントは最新の1票で上書き）
  - 管理人参加・目視監視型

2. 技術スタック
- Frontend: Next.js（App Router）, Tailwind CSS
- Backend / DB: Supabase（PostgreSQL, Storage）
- Hosting: Vercel
- Automation: GitHub Actions（毎日0時(JST)のランキング再集計バッチ）

3. 画面構成（サイトマップ）

【全体構成】
- 3-1. ホーム (home)
- 3-2. 人気キャラランキングページ (ranking)
- 3-3. キャラクター個別ページ (character)
- 3-4. 編成ランキングページ (builds)
- 3-5. 編成詳細ページ (buildId)
- 3-6. ガイドライン
- 3-7. 管理者ダッシュボード

3-1. ホーム (home)
- ページ: /
- 目的: 初見で「人気」が分かり、次に「探索」で全キャラを見せ、「話題」で盛り上がりを伝え、最後に「編成」へ流す導線を作る。
【ホームは上から4段構成】

(第1段) 人気キャラランキング
- 表示内容: キャラアイコン / キャラ名 / ★平均点 / 有効票数 / 掲示板コメント数 / 注目コメント抜粋
- 注目コメント選定ロジック:
  - ★3.0以上の投票コメント（is_latest_vote = true, is_deleted = false）から最新の1件を選出
  - 該当なしの場合: is_deleted = false の全コメント（投票+掲示板）から👍数（thumbs_up_count）が最多のコメントをフォールバック表示
- 並び順: ★平均点が高い順（降順）
- 表示件数: 上位15件まで 1-3位はヒーロー表示
- 「もっと見る」: 人気キャラランキングページ（3-2）へ遷移

(第2段) キャラクターを探す
- 検索: キャラ名テキスト検索（部分一致）
- フィルター:
  - 属性（= 性格）: 純粋（緑）/ 冷静（青）/ 狂気（赤）/ 活発（黄）/ 憂鬱（紫）
  - タイプ: 守備 / 攻撃 / 支援
  - 配置: 前列 / 中列 / 後列
  - 種族: 妖精 / 獣人 / エルフ / 精霊 / 幽霊 / 竜族 / 魔女 / ???
  - レアリティ: ★1 / ★2 / ★3
- 目的: 図鑑的にキャラを探し、個別ページへの回遊導線を作る
- 備考: ステータス値による並び替えは廃止。フィルターと検索で絞り込む形に変更

(第3段) 話題のキャラクター
- 集計期間: 直近24時間
- 指標: コメント投稿数の増加（合算）
  - 投票コメント（★あり）＋掲示板コメント（★なし）
- カウント方法: 投票コメント（★あり）も掲示板コメント（★なし）も「1件 = 1」でカウントする（is_deleted = true のコメントはカウントしない）
- 並び順: 増加数が多い順（降順）
- 表示件数: 6件
- 被りルール: 人気ランキング（第1段）に掲載中のキャラは除外して選出（被りNG）
- 備考（シンプル優先）: 集計対象外コメント（365日超え / ネットスコア -10以下）も、投稿自体は増加数にカウントする
- 連投ガード（話題カウントのみ・最小）
  - 1キャラにつき、同一 user_hash の投稿は「直近24時間で最大3件まで」増加数にカウント
  - 4件目以降も投稿・表示は可能（ただし話題カウントには加算しない）

(第4段) 編成ランキング
- 目的: キャラ単体評価（★）とは別軸で「使い方（編成）」の集合知を蓄積し、コンテンツ不足感を解消する
- 表示内容（ホーム抜粋）: 編成（キャラアイコン並び）/ コメント抜粋 / 👍/👎ボタン＋👍数 / 更新日
- 表示件数（ホーム抜粋）: 上位5件
- 導線: 「編成ランキングをすべて見る」ボタンから編成ランキングページ（3-4）へ遷移

3-2. 人気キャラランキングページ (ranking)
- ページ: /ranking
- 目的: 人気ランキングの全件を閲覧できる専用ページ
- ヘッダー表示:
  - タイトル: 「人気キャラランキング」
  - サブタイトル: 「みんなの投票と評価で順位が決まる」
  - バッジ: 「集計：毎日0時」
- 属性フィルタータブ: 全属性 / 純粋 / 冷静 / 狂気 / 活発 / 憂鬱
- 1位アナウンス: 「現在の全属性の1位は ○○」（選択中のフィルターに応じて変動）
- キャラグリッド表示:
  - キャラアイコン / 名前 / ★評価 / 投票数 / 属性バッジ / 順位番号
  - グリッドレイアウト（モバイルは3〜4列程度）
- ページネーション: 「さらに表示（残り○キャラ）」ボタン
- 注目のキャラクター（直近24時間）セクション:
  - キャラカード: アイコン / 名前 / ★ / 投票数 / コメント抜粋 / 👍数
  - ホームの「話題のキャラクター」と同じデータソース
- 票が少ないキャラの扱い: セクション5-9参照

3-3. キャラクター個別ページ (character)
- ページ: /characters/[slug]
- 構成: 性能詳細 ＋ 投稿フォーム ＋ 掲示板 ＋ 回遊エリア（同属性キャラ等）
- レイアウト: 基本情報は左右に並べて表示
- 表示優先順位:
  1) キャラ名・評価（最優先・ヒーローエリア）
  2) 愛用カード（metadata.relic: name, image_url, description, params）— PC: ★の下に展開式、モバイル: スキルの上にタップ展開
  3) 大好物（favorite_item_id → items テーブル）— PC: 右カラム、モバイル: 折りたたみ
  4) アルバイト報酬（character_rewards junction、複数対応、sort_order で並び順管理）— PC: 右カラム、モバイル: 折りたたみ
  5) スキル（PC: 2列グリッド展開、モバイル: 完全展開式）
  6) 戦闘に関わる属性・ステータス数値（展開）
- スキル表示仕様:
  - 各キャラに以下の5スロット（4カテゴリ）のスキルを表示する:
    | カテゴリ | key | スキル名 | クールタイム | 説明 | パラメータ |
    |---------|-----|---------|------------|------|-----------|
    | 低学年スキル | low_grade | あり | なし | あり | あり |
    | 高学年スキル | high_grade | あり | あり（秒） | あり | あり |
    | パッシブスキル | passive | なし | なし | あり | あり |
    | 普通攻撃（基本） | normal_attack_basic | なし | なし | あり | あり |
    | 普通攻撃（強化） | normal_attack_enhanced | なし | なし | あり | あり（任意） |
  - 普通攻撃は「普通攻撃」セクション内に基本/強化をまとめて表示。強化がなければ基本のみ表示
  - 各スキルカードの表示要素: カテゴリラベル / スキル名（低学年・高学年のみ） / クールタイム（高学年のみ） / 説明文 / パラメータ（改行区切りでリスト表示）
- 投稿フォーム:
  - 掲示板（コメント欄）の直上に静的配置
  - 単一フォーム構成（タブ切り替えなし）:
    - 名前（任意、デフォルト「名無しの教主」）
    - ★評価（任意。入力すれば投票コメント、未入力なら掲示板コメント）
    - コメント本文
- コメント一覧:
  - タブ3種: 新着順 / 👍順 / 👎順（すべて並び替え、全コメント対象）
  - デフォルト: 新着順
  - 各コメントに👍/👎リアクション（5-10参照）
  - 各コメントに通報ボタンを表示（通報対象: target_type = 'comment'）
- コメント下の回遊エリア: A（次に見る）＋B（ランキング導線）
  - Aの選定ロジックを「同属性 × 同レアリティ」主軸に固定する

3-4. 編成ランキングページ (builds)
- ページ: /builds
- ページ内で PvP / PvE / 次元の衝突 をタブ切り替え（3パターン）
- party_size（枠数）: PvP=6、PvE=6、次元の衝突=9
- ページ構成（上から）
  1) タイトル：編成ランキング
  2) 上部タブ：PvP / PvE / 次元の衝突
  3) フィルター（表示用 + 投稿支援）
     - 属性フィルター：すべて / 各属性 / 混合
     - 目的：閲覧時の絞り込み＋投稿時のキャラ選択を楽にするために必須
  4) ランキング一覧（メイン）
     - 並び順：👍が多い順（降順）
     - 初期方針：ランキング集計は「全部入り1本」
       - フィルターごとに集計を分けない（スカスカ防止・密度優先）
       - フィルターは”表示を絞るだけ”（順位基準は共通）
  5) 投稿フォーム（ページ下部）
     - 基本は「閲覧→納得→投稿」の流れを優先し、投稿圧を避ける
     - 上部に「＋編成を投稿」ボタンを置き、押すと投稿フォームへジャンプできる導線を用意

- 一覧カード表示:
  - 編成（キャラアイコン並び）
  - コメント（最大3行まで表示、以降は展開）
  - 👍/👎ボタン＋👍数
  - 日付（created_at または updated_at）
  - （任意）投稿者名、属性タグ
  - カードクリックで編成詳細ページ（3-5）へ遷移

3-5. 編成詳細ページ (buildId)
- ページ: /builds/[buildId]（buildId は UUID）
- 目的: 個別の編成について詳細閲覧・コメント交流ができるページ
- ページ構成（上から）:
  1) 編成情報ヘッダー:
     - タイトル（投稿者が付けた編成名）
     - キャラアイコン並び（編成メンバー）
     - 属性タグ
     - 投稿者名 / 日付
     - 説明文（投稿者のコメント）
     - 👍/👎ボタン＋👍数
     - 通報ボタンを表示（通報対象: target_type = 'build'）
  2) コメント投稿フォーム:
     - 名前（任意、デフォルト「名無しの教主」）
     - コメント本文
  3) コメント一覧:
     - タブ3種: 新着順 / 👍順 / 👎順
     - 各コメントに👍/👎リアクション
     - 各コメントに通報ボタンを表示（通報対象: target_type = 'build_comment'）
  4) 似ている編成セクション:
     - 同じキャラを含む / 同属性の編成を表示（回遊導線）

5. 投稿・ランキング集計ロジック

5-1. コメント種別（確定）
- コメントは2種類に分ける
  1) 投票コメント: ★評価あり（ランキング集計に使う）
  2) 掲示板コメント: ★評価なし（雑談/質問/反応用）
- 「コメントのみ（★なし）」投稿を許容する（＝掲示板コメント）
- ユーザーによるコメント編集・削除:
  - 投票コメント: 再投稿で★評価を上書き可能（5-3参照）。過去の本文は削除不可
  - 掲示板コメント: 編集・削除不可（投稿後は変更できない）
  - 編成コメント: 編集・削除不可

5-2. コメント仕様
- 投票コメント:
  - 名前: 任意（デフォルト「名無しの教主」）
  - 評価: 0.5〜5.0点（0.5刻み）
  - コメント: 最大300文字 / 最大8行
  - 同一キャラへの投稿: 最新の1票で上書き
- 掲示板コメント:
  - 名前: 任意（デフォルト「名無しの教主」）
  - コメント: 最大300文字 / 最大8行
- 編成コメント（build_comments）:
  - 名前: 任意（デフォルト「名無しの教主」）
  - コメント: 最大300文字 / 最大8行
- レートリミット（最小・確定）:
  - 投票コメント：同一キャラに対して 30秒に1回まで
  - 掲示板コメント：同一キャラに対して 10秒に1回まで
  - 編成コメント（build_comments）：同一編成に対して 10秒に1回まで

5-3. 投票コメント（★あり）の更新仕様（確定）
- 同一キャラに対して同一 user_hash が再投稿した場合：
  1) ★評価（数値）だけ更新する
     - 集計に使う★は「最新の★評価」に置き換わる（=上書き）
     - つまり「最新の★が有効」になる
  2) テキスト（投票コメント本文）は履歴として残す
     - 過去の投票コメント本文は消さない
     - 掲示板上で表示する（折り畳みしない）
- 補足（用語）
  - 「最新の★評価」＝集計対象になるもの
  - 「過去の本文」＝表示用（ランキング集計には影響しない）

5-4. ユーザー識別
- user_hash: Cookie＋IPアドレスから生成するハッシュ値
  - 生成方法:
    - サーバーサイド（API Route）で生成
    - 初回アクセス時にランダムUUIDを生成し、httpOnly Cookie に保存（有効期限: 1年）
    - user_hash = SHA-256(cookie_uuid + client_ip + server_secret)
    - server_secret は環境変数で管理
  - Cookie クリア時: 別ユーザーとして扱われる（匿名サイトのため許容）
  - 方針: 生IPをそのまま保存しない（運用・規約側で明記予定）
  - 注意: IP由来の識別子を荒らし対策目的で利用する旨を規約に明記する

5-5. 集計除外ルール（表示は維持）
【前提】
- is_latest_vote = true かつ is_deleted = false の投票のみが集計対象（過去の上書き済み投票 is_latest_vote = false、および削除済み投票 is_deleted = true は除外）
【鮮度】
- 投稿から365日が経過した投票コメントは集計から除外（表示は維持）
【品質（確定）】
- ネットスコア = 👍 − 👎
- ネットスコアが -10以下の投票コメントは集計から除外（表示は維持）
- 重要: 除外された投票コメントは「★平均計算」からも「有効票数」からも除外する

5-6. コメント数（投票数）の定義（確定）
- コメント数（投票数）＝「有効な投票コメント数」
- 有効票に含めないもの:
  1) 投稿から365日が経過して集計除外になった投票コメント
  2) ネットスコア（👍−👎）が -10以下で集計除外になった投票コメント
  3) is_latest_vote = false の投票コメント（上書き済み。5-5参照）
  4) is_deleted = true の投票コメント（管理者による削除済み。5-5参照）

5-7. 最低投票数ルール（確定）
- 有効な投票コメント数が 0件のキャラは、ランキング集計対象外
- ★平均点の計算・順位付けの対象にしない
- 1件以上からランキングに参加できる
- 判定は「有効票数のみ」で行う（集計除外票は票数に含めない）

5-8. ランキング（人気）算出
- ★平均点: 単純平均（対象＝有効な投票コメント）
- 並び順の基本: ★平均点（降順）
- 同点処理（タイブレーク）: ★が同点の場合、有効票数が多いキャラを上にする。★・有効票数がともに同点の場合: created_at が新しいキャラを上にする

5-9. 人気キャラランキングページ（3-2）の集計仕様
- 初期ソート: ★平均点（単純平均）の高い順（降順）
- 同点処理: ★同点→有効票数が多い方を上。さらに同点の場合: created_at が新しいキャラを上（5-8参照）
- 属性フィルター: 全属性 / 各属性でフィルタ可能（表示絞り込みのみ、集計基準は共通）

【票が少ないキャラの扱い（確定）】
- 有効票1以上: 通常ランキング（順位あり）として表示
- 有効票0: ランキングページ下部に「票が少ないキャラ（順位対象外）」エリアを作って表示
  - 表示はするが順位は付けない
  - 理由ラベルを明記（例：「票が少ないため順位対象外」）
  - 表示項目例: アイコン / 名前 / ★（参考） / 有効票数（0）
  - 並び順（確定）:
    - 並び順：有効票数が多い順
    - 同点時：★（参考）が高い方を上
    - さらに同点時：新しい方を上

5-10. 👍/👎（リアクション）ルール（確定）
- 対象: 投票コメント（★あり）＋掲示板コメント（★なし）の両方
- 1人は同一コメントに対して「👍か👎を1回だけ」付けられる
- 変更・取り消しは許可:
  - 👍→👎へ変更OK
  - 👎→👍へ変更OK
  - 取り消し（0に戻す）もOK
- 👍/👎は“コメント単位”で付与される（投票コメント・掲示板コメントの両方。各コメントは別のリアクション対象）

5-11. 自浄作用（見た目変化・確定）
- 判定指標: ネットスコア（👍 − 👎）
- しきい値と見た目:
  - +30以上: 金枠＋輝き演出
  - +15以上: 太字＋背景強調
  - -15以下: 文字を少し薄く（グレーアウト）
  - -30以下: 文字をさらに薄くする
- 適用範囲: キャラページのコメント（comments）、編成本体（builds）、および編成コメント（build_comments）の全てに同じルールを適用する

5-12. 掲示板（キャラ個別ページ）コメント一覧のタブ（確定）
- 対象画面: キャラクター個別ページ ＞ 掲示板（コメント一覧）
- デフォルト: 新着順
- タブ3種:
  - 新着順: 全コメントを投稿日時の新しい順に表示（並び替え）
  - 👍順: 全コメントを👍数が多い順に表示（並び替え）
  - 👎順: 全コメントを👎数が多い順に表示（並び替え）
- 同点時: 👍順・👎順で同数なら「新しい方が上」
- 並び替え/フィルター対象（確定）:
  - 表示されている全コメントを対象に行う
    （集計除外コメント：365日超え / ネットスコア -10以下 も含む）

5-13. 🧩 編成（Builds）投稿・ランキング仕様（PvP / PvE 共通・初期版）
【目的】
- 既存の核（★平均点の人気ランキング＝キャラ単体評価）とは別軸で「使い方（編成）」の集合知をUGCで蓄積する。
- 「トリッカル PvP 編成」「トリッカル PvE 編成」等の検索意図が強い入口になり得る。

【編成の単位】
- mode：PvP / PvE / 次元の衝突（タブで切替）
- party_size：枠数
  - PvP=6、PvE=6、次元の衝突=9

【投稿フォーム（必須/任意）】
- 必須:
  - members（キャラ選択：長さ=party_size）
  - comment（最大200文字）
- 任意:
  - title（編成名）
    - 未入力時は自動生成（例：「火属性PvP編成」「混合PvE編成」）
  - display_name（投稿者名）
    - 未入力時はデフォルト名「名無しの教主」を適用

【投稿時の属性ラベル（シンプル運用）】
- 編成内のキャラ属性が全部同じ：その属性（単色）
- それ以外：混合
※投稿者に複雑な選択をさせない（UGCの投稿ハードルを上げない）

【フィルター】
- 属性フィルター：すべて / 各属性 / 混合
- フィルターは表示の絞り込みのみ（集計は分割しない）
- 投稿時のキャラ選択を楽にする目的でも使用する（必須）

【ランキングの並び順】
- 並び順：👍数（likes_count）降順
  - 👎数（dislikes_count）は並び順には影響しない。👎は自浄作用（5-11の見た目変化）にのみ影響する
- 同点処理：updated_at が新しい方を上（更新で再浮上できる）

【一覧カード表示】
- 表示要素（必須）:
  - 編成（キャラアイコン並び）
  - コメント（最大3行まで表示。4行目以降は「もっと読む」で展開）
  - 👍/👎ボタン＋👍数
  - 日付（created_at または updated_at）
- 上部に「＋編成を投稿」ボタンを置き、投稿フォームへジャンプできる導線を用意する
- カードクリックで編成詳細ページ（3-5）へ遷移

【編成詳細ページのコメント仕様】
- 各編成に対してコメントを投稿できる
- コメント投稿フォーム:
  - 名前（任意、デフォルト「名無しの教主」）
  - コメント本文（必須）
- コメント一覧:
  - タブ3種: 新着順 / 👍順 / 👎順
  - 各コメントに👍/👎リアクション（キャラ側コメントと同じルール: 5-10参照）
- 似ている編成:
  - 同じキャラを含む / 同属性の編成を回遊導線として表示

【ユーザー識別・重複投稿（キャラ側思想に準拠）】
- user_hash を必須とする（荒らし対策）
- 同一 user_hash × 同一 mode で複数投稿を許可
- 同一メンバー構成（members の実 ID が一致）の重複投稿は拒否（409エラー）
- members は 9スロット固定配列（null含む）で保存し、配置位置を正確に保持する

【👍/👎（編成単位）】
- 👍/👎は “編成（build）単位” で付与する
- 1人（user_hash）は同一buildに対して「👍か👎を1回だけ」付けられる
- 変更・取り消しは許可:
  - 👍→👎へ変更OK
  - 👎→👍へ変更OK
  - 取り消し（0に戻す）もOK
- キャラ側コメントの👍/👎と同じルール（5-10参照）

【レートリミット（最小）】
- build投稿：同一 user_hash が 30秒に1回まで（投票コメントと同思想）

【0件時の表示（UX）】
- 0件時は「まだ投稿がありません」だけで終わらせず、
  - 「最初の投稿者になろう」等の誘導文
  - 「＋編成を投稿」ボタンを強めに表示
- 管理人は初期投稿を仕込み可能（運用方針として許容）

6. データベース・管理仕様

【キャラクター分類マスタ】

※ このゲームでは「属性 = 性格」。性格名がそのまま属性を兼ねる。

| 分類         | カラム         | 選択肢                              | カラー  |
|-------------|---------------|--------------------------------------|---------|
| 属性（性格） | element       | 純粋 / 冷静 / 狂気 / 活発 / 憂鬱     | 緑 / 青 / 赤 / 黄 / 紫 |
| タイプ       | role          | 守備 / 攻撃 / 支援                   | — |
| 配置         | position      | 前列 / 中列 / 後列                   | — |
| 種族         | race          | 妖精 / 獣人 / エルフ / 精霊 / 幽霊 / 竜族 / 魔女 / ??? | — |
| レアリティ   | rarity        | ★1 / ★2 / ★3                       | — |

- characters（キャラクター）:
  - id (UUID, PK)
  - slug (text, UNIQUE) — キャラ識別子（URL用）
  - name (text, NOT NULL) — キャラ名
  - rarity (text) — レアリティ
  - element (text) — 属性 = 性格（純粋/冷静/狂気/活発/憂鬱）※カラー: 緑/青/赤/黄/紫
  - role (text) — タイプ（守備/攻撃/支援）
  - race (text) — 種族（妖精/獣人/エルフ/精霊/幽霊/竜族/魔女/???）
  - position (text) — 配置（前列/中列/後列）
  - stats (jsonb) — ステータス値（HP, 物理攻撃, 魔法攻撃, 防御, 速度, クリティカル 等）
  - skills (jsonb) — スキル情報（Skill[] 配列。各要素: { category: "low_grade"|"high_grade"|"passive"|"normal_attack_basic"|"normal_attack_enhanced", name?: string, cooltime?: number, description: string, params?: string }）
  - metadata (jsonb) — 愛用カード情報（relic: { name: string, image_url: string | null, description: string, params: string }）
  - image_url (text) — キャラアイコン画像URL
  - favorite_item_id (UUID, FK → items.id, nullable) — 大好物アイテム
  - is_hidden (boolean, default false) — 非表示フラグ（サイト上に表示しないが、DB上のデータは維持）
    - is_hidden = true のキャラは全公開ページ（人気キャラランキング、キャラ検索、ホーム各セクション、character_rankings）から除外
    - builds の members に含まれている場合: アイコンをグレーアウト表示（編成自体は表示維持）
  - created_at (timestamptz)
  - updated_at (timestamptz)
  - 設計方針: skills/metadata/stats は JSONB を活用した汎用設計
  - ステータス基準: 「装備なし・ボード未解放」の最大レベル時の素の数値
  - 未入力データ: 数値欄に「未入力」と表示し、ソート時は常にリスト最下部

- items（アイテムマスタ — 大好物・アルバイト報酬の共通管理）:
  - id (UUID, PK)
  - name (text, NOT NULL) — アイテム名
  - image_url (text) — アイコン画像URL（Supabase Storage: item-icons）
  - item_type (text, NOT NULL) — 'favorite'（大好物）/ 'reward'（アルバイト報酬）
  - created_at (timestamptz)
  - updated_at (timestamptz)

- character_rewards（キャラ × アルバイト報酬 中間テーブル）:
  - character_id (UUID, FK → characters.id ON DELETE CASCADE, NOT NULL)
  - item_id (UUID, FK → items.id ON DELETE CASCADE, NOT NULL)
  - sort_order (int, NOT NULL, default 0) — 表示順（0 = 優先枠）
  - PRIMARY KEY (character_id, item_id)

- comments（投票コメント / 掲示板コメント）:
  - id (UUID, PK)
  - character_id (UUID, FK → characters.id, NOT NULL)
  - user_hash (text, NOT NULL)
  - comment_type (text, NOT NULL) — 'vote'（投票コメント）/ 'board'（掲示板コメント）
  - rating (numeric) — ★評価 0.5〜5.0（0.5刻み）。vote のみ必須、board は NULL
  - body (text) — コメント本文（最大300文字 / 最大8行）
  - is_latest_vote (boolean) — 投票コメントのみ使用（掲示板コメントは NULL）。最新の★評価を持つ行を示すフラグ（投票コメント作成時は true）
  - is_deleted (boolean, default false) — 論理削除フラグ（管理者による削除）
  - thumbs_up_count (int, default 0) — 👍数（非正規化キャッシュ）
  - thumbs_down_count (int, default 0) — 👎数（非正規化キャッシュ）
  - created_at (timestamptz)
  - updated_at (timestamptz)
  - display_name (text) — 投稿者名（任意。未入力時はデフォルト名「名無しの教主」を適用）
  - 投票の上書きルール（5-3参照）:
    - 同一 (character_id, user_hash) で comment_type='vote' の再投稿時:
      1) 既存行の is_latest_vote を false に更新
      2) 新しい行を is_latest_vote = true で作成
    - 集計クエリは WHERE is_latest_vote = true AND is_deleted = false で最新★のみ抽出
    - 過去行の body は履歴として表示維持（is_latest_vote = false でも表示対象）
  - ユニーク制約なし（同一ユーザーが同一キャラに複数投稿可能＝履歴保持のため）

- character_rankings（ランキング集計キャッシュ・バッチ更新）:
  - character_id (UUID, FK → characters.id, PK) — 1キャラ1行
  - avg_rating (numeric) — ★平均点
  - valid_votes_count (int) — 有効投票数
  - board_comments_count (int) — 掲示板コメント数
  - rank (int) — 順位（★平均点降順、同点時は有効票数降順、さらに同点時は created_at 降順）
  - updated_at (timestamptz) — 最終集計日時

- comment_reactions（コメント👍/👎）:
  - id (UUID, PK)
  - comment_id (UUID, FK → comments.id, NOT NULL)
  - user_hash (text, NOT NULL)
  - reaction_type (text, NOT NULL) — 'up' / 'down'
  - created_at (timestamptz)
  - updated_at (timestamptz)
  - UNIQUE (comment_id, user_hash) — 1人1コメントにつき1リアクション

- site_config（サイト設定・シングルトン: 1行のみ）:
  - id (int, PK, CHECK(id = 1)) — シングルトン制約
  - display_name (text) — サイト表示名
  - labels (jsonb) — ステータス項目名の動的ラベル等
  - description (text) — SEO説明文

- blacklist（ブラックリスト）:
  - id (UUID, PK)
  - user_hash (text, UNIQUE, NOT NULL) — BAN対象ユーザーのハッシュ
  - reason (text) — BAN理由（管理者メモ）
  - created_at (timestamptz)
  - 運用ルール:
    - BAN ユーザーは全投稿（コメント/編成/リアクション）を拒否する
    - 既存投稿は表示維持（削除は管理者が個別対応）
    - 投稿拒否時は汎用エラー表示（BAN されていることは明示しない）

- builds（編成投稿）:
  - id (UUID, PK)
  - mode (text, NOT NULL) — 'pvp' / 'pve' / 'dimension'
  - party_size (int, NOT NULL)
  - members ((UUID|null)[], NOT NULL) — 9スロット固定配列（配置位置を保持、空きスロットは null）
  - element_label (text) — 単色属性名 / '混合'
  - title (text) — 編成名（任意。未入力時はアプリ側で自動生成）
  - display_name (text) — 投稿者名（任意。未入力時はデフォルト名「名無しの教主」を適用）
  - comment (text, NOT NULL) — コメント（最大200文字）
  - user_hash (text, NOT NULL)
  - likes_count (int, default 0) — 👍数（非正規化キャッシュ）
  - dislikes_count (int, default 0) — 👎数（非正規化キャッシュ）
  - is_deleted (boolean, default false) — 論理削除フラグ（管理者による削除）
  - created_at (timestamptz)
  - updated_at (timestamptz)
  - 上書きルール: 同一 user_hash × 同一 mode × 同一 party_size は最新で上書き

- build_reactions（編成👍/👎）:
  - id (UUID, PK)
  - build_id (UUID, FK → builds.id, NOT NULL)
  - user_hash (text, NOT NULL)
  - reaction_type (text, NOT NULL) — 'up' / 'down'
  - created_at (timestamptz)
  - updated_at (timestamptz)
  - UNIQUE (build_id, user_hash) — 1人1編成につき1リアクション

- build_comments（編成コメント）:
  - id (UUID, PK)
  - build_id (UUID, FK → builds.id, NOT NULL)
  - user_hash (text, NOT NULL)
  - display_name (text) — 投稿者名（任意。未入力時はデフォルト名「名無しの教主」を適用）
  - body (text, NOT NULL) — コメント本文（最大300文字 / 最大8行）
  - is_deleted (boolean, default false) — 論理削除フラグ（管理者による削除）
  - thumbs_up_count (int, default 0) — 👍数（非正規化キャッシュ）
  - thumbs_down_count (int, default 0) — 👎数（非正規化キャッシュ）
  - created_at (timestamptz)
  - updated_at (timestamptz)

- build_comment_reactions（編成コメント👍/👎）:
  - id (UUID, PK)
  - build_comment_id (UUID, FK → build_comments.id, NOT NULL)
  - user_hash (text, NOT NULL)
  - reaction_type (text, NOT NULL) — 'up' / 'down'
  - created_at (timestamptz)
  - updated_at (timestamptz)
  - UNIQUE (build_comment_id, user_hash)

- reports（通報）:
  - id (UUID, PK)
  - target_type (text, NOT NULL) — 'comment' / 'build' / 'build_comment'
  - target_id (UUID, NOT NULL) — 対象レコードのID
  - user_hash (text, NOT NULL) — 通報者
  - reason (text) — 通報理由
  - status (text, default 'pending') — 'pending' / 'resolved' / 'dismissed'
  - created_at (timestamptz)
  - UNIQUE (target_type, target_id, user_hash) — 同一ユーザーによる同一対象への重複通報を防止
  - レートリミット: 同一対象に対して 60秒に1回まで

- 整合性:
  - キャラは削除しない（非表示運用）: is_hidden フラグで制御。関連データ（comments, builds等）はすべて維持
  - build削除時: 論理削除（is_deleted = true）。関連データ（build_reactions, build_comments, build_comment_reactions）は維持。サイト上は「この編成は削除されました」と表示
  - build_comment削除時: 論理削除（is_deleted = true）。関連データ（build_comment_reactions）は維持
  - 非正規化カウントの整合性:
    - comments.thumbs_up_count / thumbs_down_count: comment_reactions の INSERT/UPDATE/DELETE 時にトリガーで自動更新
    - builds.likes_count / dislikes_count: build_reactions の INSERT/UPDATE/DELETE 時にトリガーで自動更新
    - build_comments.thumbs_up_count / thumbs_down_count: build_comment_reactions の INSERT/UPDATE/DELETE 時にトリガーで自動更新
  - reports の参照整合性:
    - target_type + target_id はFK制約なし（ポリモーフィック参照）
    - 対象が削除済みの場合、管理画面で「対象が削除されています」と表示して運用対処

  - キャラアイコン画像を管理画面からアップロード
  - 保存先: Supabase Storage（バケット: character-icons）
  - next/image で配信（WebP変換・自動リサイズ）

  - 投票コメント削除時: is_latest_vote が true の行を削除した場合、その行の is_latest_vote を false に更新し、同一 (character_id, user_hash) の直前の行を true に繰り上げる

  - 必要に応じて blacklist（BAN）で user_hash を拒否

9. 運用（確定）
- 人気ランキング再集計（毎日0時 JST）:
  - GitHub Actions で Supabase Edge Function（または API Route）を呼び出し
  - 集計内容: キャラごとに以下を計算し character_rankings テーブルに保存
    - avg_rating: 有効投票の★平均点
    - valid_votes_count: 有効投票数
    - rank: 順位（★平均点降順、同点時は有効票数降順、さらに同点時は created_at 降順）
    - board_comments_count: 掲示板コメント数
  - 表示: サイトはこの集計結果テーブルを参照して表示する
- 話題キャラは直近24時間の増加数を都度算出する（結果は短時間キャッシュ推奨）
- 初期リリース時は「0件問題」回避のため、管理人が一定数の投稿（編成/投票）を仕込む運用を許容する
